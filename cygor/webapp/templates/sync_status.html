About Database Sync

The database sync feature scans your results directory and ingests all Nmap XML and module JSON files into the database. This is useful after running on-demand scans to immediately see results without restarting the web application.

What gets synced:
  • Nmap Scan Results - XML files containing host, port, and service information
  • Module Results - JSON output from enumeration modules
  • Deduplication - Existing hosts/ports are updated, duplicates are merged
When to sync:
  • After completing an on-demand scan via the Tasks page
  • After running manual scans via CLI that output to the results directory
  • When you notice scan results are not appearing in the web UI
