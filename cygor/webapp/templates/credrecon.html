About Credential Reconnaissance & Quick Guide

The Credential Reconnaissance automatically tests default and weak credentials across multiple protocols and services. It helps identify security vulnerabilities where systems use factory default or commonly-used credentials.

Supported Protocols:
  • HTTP/HTTPS: Basic Auth, Digest Auth, Form-based authentication (supports full URLs with paths)
  • SSH: Password authentication
  • FTP: Including anonymous access
  • Databases: MySQL, PostgreSQL, MS SQL Server, MongoDB, Redis
  • Remote Access: RDP (Remote Desktop), VNC (Virtual Network Computing)
  • Network: SNMP
Target Formats:
  • IP:PORT: Standard format (e.g., 192.168.1.1:22, 10.0.0.1:3306)
  • Full URLs: For HTTP/HTTPS with paths (e.g., https://example.com/admin/login.php)
  • Auto-detection: Protocol automatically detected by port number
Authorization Required: Only scan systems you own or have explicit written permission to test. Unauthorized access attempts may be illegal and violate computer fraud laws.
Format: IP:PORT or full URL with path (one per line)
