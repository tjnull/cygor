About Credential Scanner & Quick Guide

The Credential Scanner automatically tests default and weak credentials across multiple protocols and services. It helps identify security vulnerabilities where systems use factory default or commonly-used credentials.

Supported Protocols:
  • Web & Shell: HTTP/HTTPS (Basic, Digest, Form auth), SSH, Telnet, FTP
  • Databases: MySQL, PostgreSQL, MS SQL Server, MongoDB, Redis
  • Directory Services: LDAP, LDAPS (Active Directory, OpenLDAP)
  • Remote Access: RDP, VNC
  • IoT/ICS: MQTT/MQTTS (message brokers), IPMI (BMC/iLO/iDRAC)
  • Network: SNMP community string testing
Supported Input Formats:
  • IP:PORT: Standard format (e.g., 192.168.1.1:22, 10.0.0.1:3306)
  • Full URLs: For HTTP/HTTPS with paths (e.g., https://example.com/admin/login.php)
  • Auto-detection: Protocol automatically detected by port number

Smart Credential Selection

When service probing is enabled, credentials are automatically prioritized based on detected service (vendor/product). For example, if Jenkins is detected, Jenkins-specific credentials are tested first before generic HTTP credentials.

Credential Scanner Documentation

Learn how to use Cygor's credential scanner effectively.

How Credential Scanning Works
  1. Service Detection

    Identifies the service type based on port or URL, and fingerprints the target to detect vendor/product.

  2. Credential Selection

    Smart matching selects the most likely credentials based on service fingerprint and known defaults.

  3. Authentication Testing

    Tests credentials against the service using protocol-specific authentication methods.

  4. Results & Reporting

    Reports successful authentications with credential details and service information.

Command Line Options
Targets
-t, --targets List of targets (IP:PORT or URLs)
-f, --file File containing targets
-p, --protocol Force specific protocol
Credentials
--creds-file Custom credentials YAML file
--usernames Custom usernames list
--passwords Custom passwords list
--sync Sync external credential sources
Performance
--threads Number of concurrent threads
--timeout Connection timeout in seconds
--max-attempts Limit attempts per target
SSH Key Auth
--ssh-key SSH private key file for key-based auth
--ssh-key-passphrase Passphrase for encrypted private key
--ssh-cert SSH certificate file (-cert.pub) for CA-signed auth
Features
--probe / --no-probe Service probing (enabled by default)
--sync Sync external credential sources
stats Show credential database stats (subcommand)
Example Commands
Scan SSH targets with default credentials
cygor credrecon -t 192.168.1.1:22 -p ssh
Scan targets from file with auto-detection
cygor credrecon -f targets.txt --threads 20
Test HTTP login page with custom credentials
cygor credrecon -t http://target.com/login --usernames users.txt --passwords passwords.txt
Sync external sources and scan
cygor credrecon -i targets.txt --sync
Test SSH key authentication
cygor credrecon -t 192.168.1.1:22 --ssh-key ~/.ssh/id_rsa --attack-mode single --single-username root
Test with CA-signed SSH certificate (key + cert)
cygor credrecon -t 192.168.1.1:22 --ssh-key ~/.ssh/id_rsa --ssh-cert ~/.ssh/id_rsa-cert.pub --attack-mode single --single-username deploy
View credential database statistics
cygor credrecon stats
