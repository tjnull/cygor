Scan Privileges

Manage elevated privileges for scanning tools (masscan, nmap, naabu). These tools require raw socket access for network scanning.

Privilege Status
Loading privilege status...
Session Sudo Authentication

Authenticate with your sudo password to enable elevated privileges for this session. Credentials are cached and refreshed automatically until the server restarts.

Permanent Setup (Recommended)

For persistent privilege configuration that survives restarts and works in daemon mode, run the setup command from your terminal:

$ sudo cygor setup-privileges
This sets Linux capabilities and configures sudoers NOPASSWD entries for scan tools.
Linux Capabilities
Sets cap_net_raw,cap_net_admin on tool binaries. Tools run without any sudo.
Sudoers NOPASSWD
Creates /etc/sudoers.d/cygor so the web UI can run tools non-interactively.
Other commands: cygor setup-privileges status (check status), sudo cygor setup-privileges remove (undo changes)

Sync Database
About Database Sync

The database sync feature scans your results directory and ingests all Nmap XML and module JSON files into the database. This is useful after running on-demand scans to immediately see results without restarting the web application.

What gets synced:
  • Nmap Scan Results - XML files containing host, port, and service information
  • Module Results - JSON output from enumeration modules
  • Deduplication - Existing hosts/ports are updated, duplicates are merged

Proxy Configuration

These settings configure HTTP/HTTPS proxies for outbound requests. For SOCKS5 proxy or SSH tunnel routing, use the Jumpbox / Pivot settings instead.

Proxy URL for HTTP requests (e.g., http://proxy:8080 or http://user:pass@proxy:8080)
Proxy URL for HTTPS requests (usually same as HTTP proxy)
Comma-separated list of hosts/domains to bypass the proxy (e.g., localhost,127.0.0.1,.internal)
Environment Variables

Cygor also respects standard proxy environment variables if set in your shell:

HTTP_PROXY=http://proxy:8080
HTTPS_PROXY=http://proxy:8080
NO_PROXY=localhost,127.0.0.1

Settings configured above take precedence over environment variables.
